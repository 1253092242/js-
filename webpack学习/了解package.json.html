package.json

Npm 是在安装 Node. js 时附带的包管理器， Npm Script 则是 Npm 内置的一个功能，允许在 package.json 文件 里面使用 scripts 字段定义任务： 
                        {
                            "name": "webpack-test2",
                            "version": "1.0.0",
                            "main": "index.js",
                            "scripts": {
                              "test": "echo \"Error: no test specified\" && exit 1"
                            },
                            "author": "",
                            "license": "ISC",
                            "dependencies": {
                              "webpack": "^4.5.0",
                              "webpack-cli": "^3.1.0"
                            },
                            "devDependencies": {
                              "webpack": "^4.5.0"
                            },
                            "keywords": [],
                            "description": ""
                          }

scripts属性

                ” scripts ”: { 
                    ” dev”:”node dev. j s ”, 
                    ” pub”:”node build. j s ” 
                }
scripts 字段是一个对象，每个属性对应一段 Shell 脚本，以上代码定义 了两个 任务： dev 和 pub 。其底层实现原理是通过调用 Shell 去运行脚本命令，例如，执行 npm run pub 命令等同于执行 node build.j s 命令。
个人理解就是设置一个快捷方式，如运行npm run dev 就是等同于运行node build.js，暂时不知道它有什么用*-*

dependencies和dependencies

                "dependencies": {
                    "webpack": "^4.5.0",
                    "webpack-cli": "^3.1.0"
                    },
                    "devDependencies": {
                    "webpack": "^4.5.0"
                    },
我们在使用npm install 安装模块或插件的时候，有两种命令把他们写入到 package.json 文件里面去，比如： –save-dev 安装的 插件，被写入到 devDependencies 对象里面去 –save 安装的 插件 ，被写入到 dependencies 对象里面去 他们又什么区别： devDependencies 里面的插件只用于开发环境，不用于生产环境 dependencies 是需要发布到生产环境的。
简单的来说呢：就是不同的环境可以用不同的依赖和环境。
